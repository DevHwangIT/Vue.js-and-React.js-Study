React에 관한 내용 수업

컴포넌트는 쪼개진 화면이다.

컴포넌트를 사용하면 화면의 중복을 제거할 수 있다.
=> 화면의 재사용

컨테이너 컴포넌트는 데이터 보관하는 목적으로 사용되는 컴포넌트다.
=> 재사용이 어렵다

프레젠테이션 컴포넌트는 데이터의 화면 표시 사용되는 컴포넌트다.
=> 재사용이 된다.
JSX는 javaScript XML의 약자이다
리액트는 데이터로 화면을 조작한다.
=> state, props
단축키:
reactplugin ==>
rfc ==> 함수형 컴포넌트 만들 때
styled ==> 컴포넌트용 CSS 만들 때
rdxf ==> redux 코딩할 때

React -> hook이란 뭔지에 대한 설명

React에서는 JSX(Java Script XML)을 사용. => 최종적으로는 바벨에 의해서 Js로 변환됨.

개인적으로 궁금한 내용

  질문 ) 부트스트랩이란 뭔가요?
    - CSS, JS 등으로 UI를 미리 만들어 놓은 라이브러리

  질문 ) Vue Instance는 하나라고 하였는데.. 왜 여러개 인가요?
    - Vue로 프로젝트를 구성하면 하나의 Vue만으로 이를 관리하나, Vue를 plugin처럼 사용할 경우 여러개의 Vue로 관리한다
      예전에는 여러개의 Html[MPA]을 구성하였으나, 요즘에는 하나의 Html을 만든다.
      => 이러한 이유? Html을 여러개 만들면 그만큼 읽어와야되는 양이 많아지고, 트래픽이 증가하므로(트래픽=돈) 방향성이 변경되었다.

  질문 ) 만일 웹앱을 만들고 싶다면?
    - 웹페이지를 만들고나서 별도의 툴?(여러개가 있겠지..) 한번 더 랩핑해서 앱형태로 만드는 것.

  질문 ) Vue에서 엘리먼트에 넣는 id는 어떤식으로 넣어야 되나?
    - Html에서 코드제어가 필요한 영역을 태그로 한번 감싸고 이를, Vue로 빼서 처리한다.
      전부다 id로 감싸지 않고, 일부의 영역만을 감싸는 id를 사용하는 이유?
      => html에서 코드 제어가 불필요한 영역이 있을수도 있으니까

  질문 ) 웹뷰로 써도 결국 다 동작할텐데... 리액트와 리액트 네이티브가 있는이유?

    - 질문하기전 나의 생각 : 알아본 바에 의하면, 리액트 네이티브는 네이티브 언어로 재컴파일되기 때문인데, 단순 성능때문에 사용하는건가?

    - 리액트의 등장 배경을 먼저 이야기하자면
      MPA(Micro Service Archecture) 및 SPA(Single Page Application) 등 이를 보다 나은 쪽으로 개선하기 위해 등장함
      가상 돔때문에 기존 환경보다 빠르다.
      배우기가 쉬운? 혹은 접근성이 좋아서 인력이 많음. 즉, 필요한 인력 최종적으로 자본 및 프로그램 성능에 우월함.
      결국 안드로이드 앱을 만들고 싶다 아이폰을 앱을 만들고 싶다 -> 크로스 플랫폼
      자체적인 네이티브 언어로 재컴파일 되기때문에, 디바이스에서만 존재하는 해당 기능에 대한 접근이 가능할수도 있으니까
      근데 결국 리액트 네이티브나 리액트 둘 중 어떤것을 선택한다는 것에 대해서는 말하기에 애매모호하다.

  질문 ) 만일 웹 공부를 한다면 어떤 책을?
    - React의 PDF 첫 페이지에서 설명한 기초부터 완성까지 프런트 엔드 책을 추천
      리액트를 본격적으로 하고싶다면 실전 리액트 프로그래밍에 대한 책 추천

  질문 ) 웹에서는 데이터를 어떻게 관리하는가? 만일, 쇼핑몰 구조가 있다고했을 때, 최상위 부모 컴포넌트에서 이러한 데이터를 전부다 갖는건가?
    - 해당 질문 전에, 가정 여러개의 html인가? 아니면 단 하나의 html 방식인가?
      => 가정 ) 단 하나의 html이고, vue나 react로 만든 프로젝트 라는 가정하에 진행된다.
        최상위 컴포넌트가 모든 데이터를 다 갖고 있으면, 결국 너무 복잡해지기 때문에 Vuex 내부에서 여기서 관리하는데,
        이러한 것을 내부의 모듈 형태로 관리한다.
